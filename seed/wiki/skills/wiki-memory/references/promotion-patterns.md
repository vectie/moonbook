# Promotion Patterns

## Pattern: repeated blocker

If the same blocker appears across runs:

1. keep the immediate blocker in `keeper/WORKING.md`
2. summarize the recurring pattern in `keeper/MEMORY.md`
3. if the blocker teaches something about the domain, update a synthesis page

## Pattern: stable architecture insight

If a run reveals a durable architectural relationship:

1. write the maintained source page
2. revise entity or concept pages
3. keep only the operational residue in Keeper memory

## Pattern: ephemeral debug detail

If the detail only helps immediate debugging:

1. keep it in `wiki/history/debug-journal.md` or `keeper/WORKING.md`
2. do not pollute stable concept pages
