# Memory Boundaries

## Purpose

This reference defines where information should live after a run.

## Send to `keeper/MEMORY.md`

- durable operational lessons
- repeatable workflow constraints
- stable system behavior observations

## Send to `keeper/USER.md`

- stable user preferences
- collaboration style rules
- recurring presentation expectations

## Send to `keeper/WORKING.md`

- short-lived task residue
- temporary hypotheses
- in-progress coordination notes

## Send to `wiki/`

- domain knowledge
- persistent findings
- maintained source summaries
- entities, concepts, claims, and reusable answers

## Drop entirely

- empty success wording
- duplicate residue
- generic status filler
